## Goal
Obtain a critical review of repository "fluff" from Copilot.

## Background
Fluff consists of non-operational tokens, redundant documentation,
marketing-speak, and noop tokens (tokens with no meaning for the task).

Definition of Fluff:
- Non-operational tokens (comments that don't explain why, boilerplate).
- Redundant documentation (repeating AGENTS.md in every TASK.md).
- Marketing-speak (hype words like "revolutionary", "seamless").
- Noop tokens (tokens that have no meaning for the task/packet).

## Pre-requisites
- read access to repo

## Plan
1. Present the repository documentation to Copilot.
2. Ask for an identification of unnecessary boilerplate, conversational
   filler, and noop tokens based on the provided definition.
3. Record the feedback.
